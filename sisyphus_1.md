## 🏗️ 架构决策

### 1. 分层架构 (关键决策)

```
表示层 (PyQt6) 
    ↓ 回调
应用层 (Services)
    ↓ 调用
领域层 (Models)
    ← 实现
基础设施层 (Audio/Parsers/Exporters)
```

**决策理由**:
- 业务逻辑与UI完全解耦，便于测试
- 领域层纯Python，不依赖任何框架
- 基础设施可替换(音频引擎、文件格式)

### 2. 多演唱者系统 (重大设计)

**核心决策**: 演唱者切换**后台自动管理**，用户无感知

- 用户只需按歌词文本顺序连续打轴(Space)
- 系统维护全局Checkpoint序列跨所有演唱者
- 自动识别当前Checkpoint所属演唱者并渲染
- 播放时自动并排渲染所有演唱者(和声场景)

**设计原因**:
- 避免用户手动切换，保持打轴流畅性
- 支持自然歌词输入(主歌+和声混合)
- 播放时自动处理和声显示

### 3. 时间标签系统

**关键决策**: **允许时间倒退**，仅标记异常不阻止

- 用户修改时可能出现时间倒退(正常)
- 和声场景下不同演唱者时间交错(正常)
- 仅标记为"时间异常"供后续调整
- 播放时正常渲染，异常段落完成后高亮提示

### 4. 打轴交互 (核心体验)

**普通字符**: Space按下即记录时间，立即跳转下一个
**句尾字符**: 按下=开始时间，抬起=结束时间，然后跳转下一行
**双击Checkpoint**: 跳转到该时间前X秒(预听，X可配置)
**快捷键**: +/- 增删Checkpoint，L切换句尾标记

**设计理由**:
- 按下即记录: 实现连续快速打轴，无需等待抬起
- 句尾长按: 处理拖音/长音场景，记录完整演唱时长
- 双击预听: 提前听节奏，提高打轴精度

## 🔧 技术选型

### 确定选型
- **GUI框架**: PyQt6 + PyQt-Fluent-Widgets (现代化Fluent Design)
- **音频引擎**: sounddevice + soundfile (低延迟，跨平台)
- **日语处理**: pykakasi (汉字转假名注音)
- **架构模式**: 分层架构 + 回调机制

### 选型原因
- Qt: 低延迟、键盘快捷键无限制(F1-F12)、高性能渲染
- sounddevice: PortAudio后端，延迟<20ms，适合精准打轴
- Fluent-Widgets: 现代化UI，暗色主题适合长时间使用

## 📐 关键设计细节

### 1. CheckpointConfig 设计 (已修正)

```python
# 允许 check_count = 0 表示连打
check_count: int  # 默认1，可以为0

# check_count = 0 表示该字符和上一个字符连唱
# 示例: 「赤」(2个节奏点) + 「ー」(0个，连唱)
```

**经验教训**: 最初设计约束每个字符至少1个Checkpoint，后修正为允许0个，支持连打场景。

### 2. 时间格式 (已修正)

**错误示例**: `[10][12]` 过于简化，易误解
**正确格式**: `[00:10.00][00:12.00]` (分:秒.毫秒)
**存储格式**: 毫秒整数 (如 10000 = 10.00秒)

### 3. 播放渲染逻辑

**逐帧计算(60fps)**:
- 普通字符: 当前时间 - Checkpoint时间 = 渐变进度
- 连打字符: 多个Checkpoint独立计算各自进度
- 句尾拖音: 开始时间~结束时间 = 拖音进度条
- 连打字符(check_count=0): 同步于上一个字符
- 多演唱者: 每个演唱者独立计算进度

### 4. 启动流程设计

```
StartupInterface (启动界面)
├── LyricInputPanel (粘贴/导入歌词)
├── ImportPreview (自动分析节奏点+注音)
└── AudioSelectPanel (选择音频)
    ↓ 创建项目
EditorInterface (编辑器)
├── KaraokePreview (多行显示+ワイプ效果)
├── TransportBar (播放控制)
└── TimelineWidget (时间轴)
```

## 🎨 UI设计要点

### 1. KaraokePreview 多行显示

**不是仅显示当前一句**，而是像文本编辑器:
- 显示当前Checkpoint前后±N行(可配置，默认±3行)
- 允许用户看到上下文(上一句/下一句)
- Alt+滚轮缩放(50%~200%)

### 2. 视觉设计

- **暗色主题**: #202020背景，适合长时间使用
- **演唱者颜色**: 红/蓝/绿/黄区分不同演唱者
- **ワイプ效果**: 渐变高亮(已唱→正在唱→未唱)
- **时间异常**: 红色边框闪烁提示

### 3. 设置界面

**TimingSettingsPanel**:
- 自动检查配置(促音/长音/英文模式)
- 偏移量(整体/Z/X步长/打轴偏移)
- 双击预听提前量X(默认2秒)
- 显示上下文行数

## 🚨 设计陷阱与解决方案

### 陷阱1: 演唱者手动切换
**初始设计**: 用户手动切换演唱者
**问题**: 打断打轴流程，用户体验差
**解决方案**: 后台自动管理，用户无感知切换

### 陷阱2: 时间标签强制递增
**初始设计**: 强制时间标签递增，否则阻止添加
**问题**: 用户修改时必然出现倒退，阻止操作不合理
**解决方案**: 允许倒退，仅标记异常，不阻止操作

### 陷阱3: Checkpoint不能为0
**初始设计**: 每个字符至少1个Checkpoint
**问题**: 不支持连打场景(如长音"ー")
**解决方案**: 允许check_count=0，表示和上一个字符连唱

### 陷阱4: 时间格式简化
**初始设计**: 使用[10][12]简化表示
**问题**: 易误解为秒，实际存储是毫秒
**解决方案**: 使用[00:10.00]明确格式，并注明存储格式
