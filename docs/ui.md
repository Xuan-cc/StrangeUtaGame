# UI 层设计

## 概述

UI 层基于 **PyQt6 + PyQt-Fluent-Widgets**，负责用户界面展示和输入处理。本层**不包含业务逻辑**，所有业务操作委托给应用服务层。

## 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| GUI 框架 | PyQt6 | 成熟稳定，功能完善 |
| 组件库 | PyQt-Fluent-Widgets | 现代化 Fluent Design 风格 |
| 样式方案 | QSS | 类似 CSS，易于维护 |
| 图表/波形 | 自定义绘制 | 低延迟，高度可控 |

## 核心组件

### MainWindow

**职责**：
- 应用主容器
- 导航管理
- 子界面切换
- 全局快捷键处理

**启动流程**：
1. 应用启动时显示 StartupInterface
2. 用户输入/导入歌词并选择音频
3. 点击"创建项目"后切换到 EditorInterface
4. 左侧导航栏在 StartupInterface 时隐藏或禁用

**与下层交互**：
- 持有 ApplicationContext（包含各 Service）
- 响应 Service 回调更新界面

### StartupInterface（启动界面）

**职责**：
- 歌词初始输入和导入
- 音频文件选择
- 项目创建入口

**布局**：

```
StartupInterface
├── 顶部：Logo + 标题
├── 左侧：LyricInputPanel（歌词输入区）
│   ├── 大文本编辑框
│   │   - 支持多行输入
│   │   - 支持直接粘贴
│   │   - 占位符："在此粘贴歌词，或拖拽歌词文件到此处..."
│   ├── 导入按钮组
│   │   - "从文件导入 (TXT/LRC/KRA)"
│   │   - "从剪贴板粘贴"
│   │   - "清空"
│   └── 文件拖拽区域
│       - 高亮显示拖放区域
```

**核心功能**：

1. **歌词输入**
   - 直接粘贴：从剪贴板粘贴任意文本
   - 文件导入：支持 TXT（纯文本）、LRC（含时间标签）、KRA（卡拉OK格式）
   - 拖拽导入：拖放文件到指定区域

2. **智能预处理**
   - 自动分割字符（日文/英文/符号）
   - 自动分析注音
   - 自动计算检查数（节奏点数量）
   - 识别句尾（行末字符）

3. **预览与修正**
   - 显示解析后的歌词结构
   - 显示每个字符的检查数
   - 显示句尾标记
   - 支持在预览中快速编辑

4. **音频选择**
   - 选择音频文件（MP3, WAV, FLAC, AAC 等）
   - 验证音频有效性
   - 显示音频基本信息

5. **创建项目**
   - 验证歌词和音频都已提供
   - 生成 RLF 项目文件
   - 切换到 EditorInterface

**与下层交互**：
- 调用 ProjectService.create_new_project()
- 调用 AutoCheckService 分析歌词
- 调用 FileParsers 解析导入的歌词文件

### EditorInterface

**职责**：
- 打轴主界面
- 键盘事件捕获
- 协调预览组件和播放控制

**主要区域**：

1. **KaraokePreview（歌词预览）**
   - 显示多行歌词
   - 字符级ワイプ效果（渐变色）
   - 根据播放位置高亮当前字符
   - 60fps 刷新

2. **TransportBar（播放控制）**
   - 播放/暂停/停止
   - 进度条（可拖拽跳转）
   - 当前时间显示
   - 播放速度调节（0.5x ~ 2.0x）

3. **TimelineWidget（时间轴）**
   - 音频波形显示
   - 时间标签位置标记
   - 播放位置游标
   - 支持缩放

**快捷键绑定**：

| 按键 | 功能 |
|------|------|
| **Space 按下** | **【核心】为当前 Checkpoint 记录开始时间，立即切换到下一个 Checkpoint** |
| **Space 长按** | **句尾字符专用：按下开始，抬起结束句尾** |
| F1-F9 | 功能键（可配置，默认与 Space 相同） |
| A | 播放/暂停 |
| S | 停止 |
| Z | 后退 5 秒（可配置） |
| X | 前进 5 秒（可配置） |
| Q | 减速 |
| W | 加速 |
| ESC | 清除当前时间标签 |
| **+** | **在当前字符后添加节奏点（Checkpoint）** |
| **-** | **删除当前字符的最后一个节奏点** |
| **L** | **切换当前字符的句尾标记（Line End）** |
| **双击 Checkpoint** | **跳转到该时间前 X 秒（用于预听，X 可配置）** |
| **Alt + 滚轮** | **缩放界面（放大/缩小歌词显示）** |

**基础打轴交互说明**：

**Checkpoint 定义**：
- Checkpoint 表示"该字符开始播放"的时间点
- 当音频播放到 Checkpoint 的时间点时，对应的字符应该开始显示（高亮）

**普通字符打轴流程（Space 按下即完成）**：
1. 用户听到该唱某个字了
2. 按下 Space 键的瞬间，记录当前音频时间为该字符 Checkpoint 的时间
3. **立即**自动移动到下一个 Checkpoint
4. 用户无需抬起 Space 键，直接准备打下一个字

**示例**：
```
歌词：赤い花
Checkpoint：赤(1) い(1) 花(1)

播放进度：
[10.0秒] 用户听到"赤" → 按下 Space → 记录"赤"的时间为 10.0秒 → 光标自动跳到"い"
[10.5秒] 用户听到"い" → 按下 Space → 记录"い"的时间为 10.5秒 → 光标自动跳到"花"
[11.0秒] 用户听到"花" → 按下 Space → 记录"花"的时间为 11.0秒 → 完成
```

**关键点**：
- **按下即记录，无需等待抬起**
- **记录后立即跳转**，实现连续快速打轴
- 句尾字符除外（需要长按记录拖音）

### KaraokePreview 设计

**布局策略（文本编辑器模式）**：

**多行显示**：
- **不是仅显示当前播放的一句**，而是像文本编辑器一样显示多行歌词
- 显示范围：当前 Checkpoint 所在行及其前后若干行（如 ±3 行）
- 允许用户看到上下文：上一句和下一句歌词
- 滚动：自动滚动保持当前 Checkpoint 在可视区域中央

**缩放功能**：
- **Alt + 滚轮**：放大/缩小歌词显示
- 缩放范围：50% ~ 200%
- 仅缩放歌词区域，不影响控件大小
- 记住用户偏好

**ワイプ效果**：

- **未唱字符**：灰色
- **已唱字符**：高亮颜色（红色/青色）
- **正在唱字符**：从左到右渐变

**实现方式**：
- 每个字符独立 QLabel
- 自定义 paintEvent 实现渐变
- 使用裁剪区域实现部分着色
- 多行布局使用 QVBoxLayout + QHBoxLayout 组合

**刷新策略**：
- 位置变化时更新
- 60fps 刷新率
- 仅重绘变化的字符

**交互设计**：

**1. 双击 Checkpoint 跳转（预听功能）**

**用途**：快速跳转到要打轴的位置前 X 秒，提前听一下节奏

**操作**：
- 在 KaraokePreview 中双击某个 Checkpoint（节奏点标记）
- 系统自动：
  1. 获取该 Checkpoint 已记录的时间（如果已有时间标签）
  2. 或获取该 Checkpoint 的预估时间（基于前后已有时间标签插值）
  3. 跳转到：该时间 - X 秒（X 可配置，默认 2 秒）
  4. 自动播放

**配置项（在 TimingSettingsPanel 中）**：
- `double_click_pre_listen_seconds`：双击预听提前量（默认 2 秒，范围 0.5 ~ 10 秒）

**示例**：
```
当前状态：
赤[●]い花が咲いた      ← [●] 表示已打轴的 Checkpoint
    ↑
    正在打这个位置

用户双击下一个 Checkpoint（い）：
假设「い」的预估时间是 00:15.000
设置 X = 2 秒
跳转位置：00:15.000 - 00:02.000 = 00:13.000
音频跳转到 13 秒开始播放，用户听到「赤」后开始按空格打「い」
```

**2. 快速添加/删除 Checkpoint（节奏点）**

**添加 Checkpoint（+ 键）**：
- **场景**：自动检查计算的节奏点数量不够，需要手动添加
- **操作**：按 **+** 键
- **效果**：
  1. 在当前字符后插入一个新的 Checkpoint
  2. 当前字符的 check_count + 1
  3. 自动分配新的节奏点索引
  4. 光标移动到新添加的 Checkpoint
- **注意**：仅调整节奏点配置，不影响已打的时间标签（除非已有时间标签数量超过新的 check_count）

**删除 Checkpoint（- 键）**：
- **场景**：节奏点太多，需要减少
- **操作**：按 **-** 键
- **效果**：
  1. 删除当前字符的最后一个 Checkpoint
  2. 当前字符的 check_count - 1
  3. 如果该 Checkpoint 已有时间标签，一并删除
  4. 如果当前光标在该 Checkpoint，移动到前一个
- **约束**：每个字符至少保留 1 个 Checkpoint（不能全部删除）

**3. 句尾字符长按打轴（特殊交互）**

**设计目的**：句尾字符通常有拖音或长音，需要记录开始和结束两个时间点

**操作方式**：
1. **按下 Space**：记录句尾字符的开始时间（第一个 Checkpoint）
   - 此时字符开始高亮（ワイプ效果开始）
   - 保持按下状态，表示"正在唱这个句尾"
2. **保持按下**：用户听到句尾唱完（拖音结束）
3. **抬起 Space**：记录句尾字符的结束时间（第二个 Checkpoint）
   - 此时ワイプ效果完成
   - 自动切换到下一行的第一个字符

**技术实现**：
- 检测键盘按下（KeyDown）和抬起（KeyUp）事件
- KeyDown：如果当前字符是句尾（is_line_end = true），记录开始时间
- KeyUp：如果已记录开始时间，则记录结束时间，并跳转下一行

**视觉反馈**：
- 按下期间：句尾字符持续高亮（或显示进度条表示拖音进度）
- 抬起后：ワイプ完成，切换到下一行

**示例**：
```
歌词：  赤い花が咲いた
句尾：          ↑「た」是句尾

用户操作：
1. 播放到「た」时按下 Space
   - 记录时间：00:20.000（句尾开始）
   - 「た」开始高亮
2. 保持 Space 按下，听到「た」拖音结束（约 0.5 秒）
3. 抬起 Space
   - 记录时间：00:20.500（句尾结束）
   - 「た」完全高亮
   - 自动跳到下一行「空が青い」的第一个字
```

**特殊情况处理**：
- 如果用户按下后很短时间就抬起（< 100ms）：视为普通点击，只记录一个时间点
- 如果用户长按超过 5 秒：自动结束（防止误操作）
- 如果用户切换了播放位置：取消当前的按下状态

**4. 快速切换句尾标记（L 键）**

**用途**：快速标记/取消标记句尾字符（影响渲染和导出）

**操作**：按 **L** 键（Line End）
- **当前字符标记为句尾**：设置 is_line_end = true
  - 视觉效果：字符后显示句尾标记（如「|」或特殊下划线）
  - 影响：导出时可能特殊处理（如添加休止时间）
- **当前字符取消句尾**：设置 is_line_end = false
- **切换**：如果已经是句尾则取消，如果不是则标记

**句尾标记的视觉反馈**：
- 句尾字符显示特殊标记（如竖线 | 或下划线）
- 颜色略有不同（如更深的颜色）
- 行末自动视为句尾（无需手动标记）

**快捷键配置**：
所有上述快捷键（+/-/L）可在 ShortcutSettingsPanel 中自定义

### SingerManagerInterface（演唱者管理界面）

**职责**：
- 配置演唱者属性（名称、颜色）
- 添加/删除演唱者
- 分配歌词给演唱者
- **注意**：用户不需要在此界面"切换当前演唱者"，系统自动管理

**界面布局**：
```
SingerManagerInterface
├── SingerListPanel（演唱者列表）
│   ├── 演唱者列表
│   │   - 复选框（启用/禁用，禁用的演唱者不参与打轴）
│   │   - 颜色块（可点击修改，用于视觉区分）
│   │   - 名称（可编辑，如"初音ミク"、"合唱"）
│   │   - 歌词范围显示（如"第3-5行"）
│   │   - 删除按钮（至少保留一个）
│   └── 操作按钮
│       - "添加演唱者"（自动生成颜色，用于和声）
│
└── SingerDetailPanel（演唱者详情）
    ├── 名称输入框
    ├── 颜色选择器
    ├── 歌词范围配置
    │   - 起始行 / 结束行
    │   - 或：从文本中选择段落分配给该演唱者
    ├── 统计信息
    │   - 歌词行数
    │   - 已打轴/未打轴字符数
    └── 操作按钮
        - "清空歌词"（删除该演唱者的所有歌词）
        - "合并到其他演唱者"（转移后删除该演唱者）
```

**交互设计**：
- **仅配置**：此界面仅用于配置演唱者属性，不用于"切换当前"
- 点击列表项：查看/编辑该演唱者的配置
- 拖拽选择文本：在导入歌词时分配段落给演唱者

**使用场景**：
- **启动时**：默认只有"演唱者1"，无需管理
- **添加和声时**：
  1. 在 StartupInterface 输入完整歌词
  2. 选择主歌部分分配给演唱者1
  3. 点击"添加演唱者"创建演唱者2
  4. 选择和声部分分配给演唱者2
  5. 系统后台自动管理两个演唱者的顺序
- **打轴时**：用户按空格连续打轴，系统自动在演唱者间切换
- **导出时**：可选择导出全部演唱者或指定演唱者

**后台自动管理**：
- 系统维护全局 Checkpoint 序列
- 按歌词文本顺序跨演唱者排列
- 示例序列：演唱者1行1 → 演唱者1行2 → 演唱者2行1 → 演唱者1行3...
- 用户无感知，只需按空格继续

### SettingsInterface（设置界面详细设计）

**职责**：
- 应用配置管理
- 打轴参数设置
- 显示偏好设置
- 快捷键自定义

**详细设置项**：

#### 1. TimingSettingsPanel（打轴设置）

**自动检查配置**：
- 促音「っ」计数开关（是否计入节奏点）
- 长音「ー」计数开关
- 行首/行尾自动打轴开关
- 英文单词节奏点模式（单字/整词）
- 自动检查触发按钮（"重新分析当前歌词"）

**偏移量设置**：
- **整体偏移**：调整所有时间标签的基准时间（毫秒）
  - 用途：歌词整体提前或延后
  - 输入框：可输入正负值
  - 按钮：应用/重置
- **快进/快退步长（Z/X 键）**：
  - 默认值：5000ms（5秒）
  - 可调范围：100ms ~ 30000ms
  - 支持多档（Z/X 默认，Shift+Z/X 微调）

**打轴偏移**：
- 提前量/延后量：打轴时自动加减的时间（毫秒）
  - 用于补偿反应延迟
  - 通常设置为 -15ms ~ -50ms

**双击预听设置**（新增）：
- **预听提前量 X**：双击 Checkpoint 跳转时提前的秒数
  - 默认值：2 秒
  - 可调范围：0.5 ~ 10 秒
  - 用途：跳转到要打轴位置前 X 秒，提前听节奏
  - 示例：Checkpoint 预估时间为 15秒，X=2秒，则跳转到 13秒开始播放

**渲染显示设置**（新增）：
- **显示上下文行数**：KaraokePreview 显示当前 Checkpoint 前后多少行
  - 默认值：3 行（即显示 ±3 行，共 7 行）
  - 可调范围：1 ~ 10 行
  - 用途：让用户看到上下文歌词

#### 2. DisplaySettingsPanel（显示设置）

**缩放设置**（新增）：
- **歌词缩放比例**：Alt+滚轮的缩放基准
  - 默认值：100%
  - 可调范围：50% ~ 200%
  - 步进：10%
- **界面缩放**：整体界面缩放（包括控件）
  - 默认值：100%
  - 可调范围：80% ~ 150%

**主题**：
- 暗色/亮色/跟随系统
- 强调色选择

**字体**：
- 歌词字体/大小
- 界面字体/大小

**预览效果**：
- ワイプ颜色配置
- 已唱/正在唱/未唱颜色
- 渐变速度

#### 3. AudioSettingsPanel（音频设置）

**播放设备**：
- 输出设备选择
- 采样率设置

**性能**：
- 缓冲区大小（影响延迟）
- 默认播放速度

#### 4. ShortcutSettingsPanel（快捷键设置）

**可配置快捷键**：

**基础打轴键**（可修改）：
- 打轴键：默认 Space
- 添加 Checkpoint：默认 **+**
- 删除 Checkpoint：默认 **-**
- 切换句尾标记：默认 **L**
- 播放/暂停：默认 A
- 停止：默认 S
- 快进/快退：默认 Z/X
- 减速/加速：默认 Q/W

**功能键 F1-F9**（可分配给特定操作）：
- 默认：均映射为"打轴"（与 Space 相同）
- 可自定义：如 F1 设为"添加 Checkpoint"，F2 设为"切换句尾"等

**组合键**：
- Shift+Z/X：微调快进/快退
- Alt+滚轮：缩放歌词
- Ctrl+滚轮：缩放界面

**冲突检测**：
- 检测快捷键冲突
- 提示用户重新设置

### KaraokePreview 多演唱者渲染设计

**职责**：
- 同时渲染多个演唱者的歌词
- 根据播放位置高亮每个演唱者的当前字符
- 使用时间异常检测和特殊渲染

**渲染策略**：

#### 1. 自动演唱者模式（默认）
```
┌─────────────────────────────┐
│ 赤い花が咲いた                │ ← 自动显示当前 Checkpoint 所属演唱者
│     ↑（高亮，颜色自动变化）   │
└─────────────────────────────┘
```
- **核心设计**：用户不需要知道当前是哪个演唱者
- 系统自动根据当前 Checkpoint 位置显示对应演唱者
- 演唱者切换平滑，用户感知为"连续往下打"
- 使用演唱者颜色进行视觉区分
- **示例流程**：
  1. 当前是演唱者1的 Checkpoint → 显示演唱者1歌词（红色）
  2. 用户打完，下一个是演唱者2的 Checkpoint → 自动切换到演唱者2歌词（蓝色）
  3. 用户继续按空格，无感知切换

#### 2. 播放时和声渲染（自动）
```
┌─────────────────────────────┐
│ 赤い花が咲いた                │ ← 主唱（演唱者1，红色）
│     ↑                        │
│ 青い空が広がる                │ ← 和声（演唱者2，蓝色）
│     ↑                        │ ← 同时高亮
└─────────────────────────────┘
```
- **播放时自动检测**：同一时间点多个演唱者有歌词
- **自动并排渲染**：垂直排列，不同颜色区分
- **用户无需切换**：播放时自动显示所有演唱者
- **打轴时专注**：打轴时只显示当前 Checkpoint 对应的演唱者

#### 3. 时间异常渲染（用户修改导致的时间倒退）

**检测逻辑**：
- 仅在同演唱者内检测时间顺序
- 当后一个时间标签早于前一个时，标记为"异常"

**渲染方式**：
- **播放前**：正常显示，但时间标签位置显示警告图标
- **播放中**：正常高亮
- **播放后**（该段已唱完）：
  - 异常段落高亮显示（如红色背景闪烁）
  - 显示警告提示："时间异常，请调整"
  - 点击可快速跳转到该位置

**示例**：
```
时间点 1: [10秒] 赤（正常，绿色）
时间点 2: [12秒] い（正常，绿色）
时间点 3: [11秒] 花（异常，红色闪烁）← 11秒比前一个12秒早
时间点 4: [15秒] が（正常，等待播放）
```

**不阻止打轴**：
- 用户可以继续打后面的轴点
- 系统仅渲染警告，不弹出阻塞提示
- 用户完成后统一调整异常段落

#### 4. 和声场景渲染（重点）

**场景**：两个演唱者同时唱不同歌词

```
时间点: [10秒]
演唱者1: 赤い花（"赤い"正在唱，高亮红色）
演唱者2: 青い空（"青い"正在唱，高亮蓝色）
        ↑ 同时高亮，不冲突
```

**渲染实现**：
- 每个演唱者独立的渲染上下文
- 独立计算每个演唱者的当前字符位置
- 使用不同颜色区分（如红/蓝/绿/黄）
- 歌词行垂直排列，避免重叠

**颜色分配**：
- 演唱者1：红色 (#FF6B6B)
- 演唱者2：蓝色 (#4ECDC4)
- 演唱者3：绿色 (#95E1D3)
- 演唱者4：黄色 (#FCE38A)
- 更多演唱者：循环或用户自定义

## 与下层交互

### 调用应用服务

UI 层通过调用 Application Service 方法执行业务操作：

```
用户按空格键
    ↓
EditorInterface._on_space_pressed()
    ↓
TimingService.on_timing_key("SPACE")
    ↓
业务逻辑执行...
```

### 响应服务回调

服务层通过**回调函数**通知 UI 状态变更：

```
TimingService 添加时间标签
    ↓
调用 on_timetag_added 回调
    ↓
EditorInterface._on_timetag_added()
    ↓
更新 KaraokePreview 显示
```

**回调类型**：
- 数据变更回调（时间标签添加、位置变化）
- 状态变更回调（播放/暂停、项目加载）
- 错误回调（操作失败）

### 避免直接依赖

**原则**：
- UI 不直接操作领域对象
- UI 不直接调用基础设施
- 所有操作通过 Service 层

## 状态管理

### UI 本地状态

- 当前激活的界面
- 窗口大小/位置
- 用户偏好设置（主题、字体等）

### 业务状态

- 当前项目数据：存储在 ProjectService
- 打轴状态：存储在 TimingService
- UI 只读取，不直接修改

### 状态同步

- 通过回调机制同步
- 避免轮询
- 批量更新减少重绘

## 性能优化

### 渲染优化

- **双缓冲**：避免绘制闪烁
- **局部重绘**：仅更新变化的区域
- **硬件加速**：利用 GPU 加速（Qt 自动）

### 大文件处理

- **虚拟列表**：仅渲染可见的歌词行
- **波形缓存**：预生成或按需生成
- **延迟加载**：非激活界面不渲染

### 响应速度

- **异步操作**：文件 I/O 不阻塞 UI
- **进度反馈**：长时间操作显示进度
- **取消机制**：支持取消耗时操作
