# UI 层设计

StrangeUtaGame 提供了三套主要的编辑界面，分别针对打轴、行编辑和全文本编辑任务。

## 核心界面

### 1. EditorInterface (打轴界面)
这是主要的时间标签编辑界面。

- **职责**：
    - 提供类似节奏游戏的打轴体验。
    - 实时预览卡拉OK走字效果 (KaraokePreview)。
    - 全键盘驱动，支持高频率的时间标签记录。
    - **核心特性**：
        - 逐字高亮渲染，支持 Ruby 同步走字。
        - **无时间戳字符连读渲染**：支持有 checkpoint 但无时间戳的字符，参照邻近组的时间进行连续平滑渲染。
        - 连词 (Linked Words) 渲染。
        - **全局 Offset 调节**：工具栏 QLineEdit 输入框实时调整并保存全局偏移（支持输入 -2000~2000 范围的整数值），联动更新 Character 的 `render_timestamps`。
        - **单击跳转**：单击字符直接将打轴目标移动到该字符的第一个 checkpoint。
        - 实时应用时间偏移。

### 2. EditInterface (行编辑界面)
提供基于表格的项目全局视图。

- **职责**：
    - 展示所有歌词行及其元数据（行号、演唱者、连词标记等）。
    - **行级操作**：添加行、删除行、复制行、插入行，支持多行选择。
    - **键盘快捷键**：Ctrl+C 复制行、Ctrl+V 粘贴行、Delete 删除行。
    - **LineDetailDialog**：双击行打开详细编辑对话框，可逐个 Character 编辑注音、节奏点、句尾标记（is_sentence_end）和演唱者。
    - **批量变更对话框**：支持多字符划选自动填充注音；移除"将匹配词设为连词"复选框，变更为继承原字符的连词属性。
    - **字符级操作**：在行详情中支持添加、删除、复制、插入字符，Ctrl+C/V/Delete 快捷键。
    - 操作后自动维护领域不变量（is_line_end、is_sentence_end、check_count、linked_to_next）。句尾字符允许 check_count=0（无普通节奏点时以前一个字符的 checkpoint 为开始）。

### 3. RubyInterface (全文本编辑界面)
专注于注音编辑和文本调整。

- **职责**：
    - **全类型注音**：自动分析为所有字符类型（汉字、平假名、片假名、英文、数字、符号、空格）生成注音。
    - 用户可通过「按类型删除注音」按钮按字符类型批量清除不需要的注音（含長音符号ー/～、特殊符号♪、小假名如 ぁぃぅぇぉゃゅょゎっ等）。
    - 手动编辑注音文本及其对应的字符范围。
    - **自由文本编辑**：支持直接修改歌词行文本，系统自动维护已有的时间标签和注音。
    - **连词编辑格式**：全文本编辑采用 `{text|r1,r2,...}` 格式表示连词。各字符按 `linked_to_next` 链条分组，各字独立读音。例如 `{大冒険|だい,ぼう,けん}` 表示三字词各自注音，`{One|ワン,,}` 表示英文词注音位置。向后兼容旧版 `漢字{かんじ}` 格式。
    - **自动应用修改**：切换标签页离开全文本编辑时自动将修改应用回项目数据，无需手动点击按钮。
    - **注音保留**：文本未变更时仅更新标注的注音，不清除未标注字符的已有注音；文本变更时通过 SequenceMatcher 映射保留匹配字符的注音。
    - **更新节奏点**：根据用户调整后的注音数据和 AutoCheck 设置规则重新计算节奏点。更新前自动保存全文本编辑框中的内容。

## 辅助界面
- **MainWindow**：基于 FluentWindow 的主框架，包含侧边栏导航。全局 Ctrl+S 快捷键保存项目（已有路径直接保存，无路径弹出另存为对话框）。切换标签页时自动应用全文本编辑界面的修改。启动时检查闪退恢复文件，询问用户是否加载。未保存修改关闭时，使用中文按钮（保存/放弃/取消）提示。
- **StartupInterface**：项目创建向导，负责初始歌词导入和音频选择。
- **HomeInterface**：主页，管理歌词导入和快捷入口。
    - **歌词导入流程**：导入文件或拖入文件时，仅读取原始文件内容显示到文本框，不进行解析。格式解析延迟到点击「创建项目」时执行。
    - **格式自动检测**：支持从文本内容自动识别 ASS、SRT、LRC（逐行/逐字/增强型）、Nicokara、内联格式和纯文本。
    - **支持导入格式**：LRC、ASS、SRT、TXT、KRA 文件。
    - 拖拽支持 .lrc/.txt/.kra/.ass/.srt 歌词文件和 .mp3/.wav/.flac 音频文件。
- **ExportInterface**：导出配置界面。
    - **演唱者过滤改进**：
        1. 自动过滤：仅显示当前打轴模式中实际使用的演唱者，无需手动排除。
        2. 可滚动列表：演唱者过滤器使用 QScrollArea，支持滚轮操作，避免演唱者过多时溢出。
        3. 演唱者标签独立设置："在演唱者切换处插入【演唱者名】标签"选项与演唱者过滤无关，作为单独设置项放在导出设置中。

### 4. SettingsInterface (设置界面)
管理全局配置与校准工具。

- **职责**：
    - **Offset 校准（可视化弹窗）**：
        - 点击「开始校准」弹出独立校准窗口。
        - 深色画布上 2 个白色滑块按 BPM 从左到右匀速滑动。滑块中点（视觉正中央）穿过中央红色判定线对应一拍，接近判定线时自动放大。
        - 滑块穿过判定线时播放节拍器点击音（使用 `sd.OutputStream` 实时生成，提供更精准的时间同步）。
        - 精确的 spin-wait 计时机制，确保亚毫秒级（sub-ms）的定时精度。
        - 用户按空格键敲击，系统以判定线中点为完美判定时刻计算偏移量。
        - 左上角实时显示「最近偏移」和「平均偏移」。
        - 右上角「重置」和「应用」按钮，应用后写入全局 Offset 设置。
        - BPM 可在校准过程中自由调节（60-240），动态影响判定窗口。
        - 窗口关闭或隐藏时完全销毁并清理节拍器资源。`_on_apply` 和 `_on_reset` 后直接关闭窗口时，通过在 `accept()` 前主动调用 `_stop_metronome()` 确保音频不再残留。
    - **打轴设定**：
        - 默认打轴偏移（tag_offset_ms）设为 -130ms。
        - Karaoke 渲染以及导出偏移：-100ms。
    - **Auto Check 设置**：新增小写假名 check 开关、句尾判定开关，控制自动分析逻辑。
    - **配置管理（About 界面）**：
        - 提供「打开目录」按钮，快速访问配置文件夹。
        - 提供「更改位置」按钮，允许用户自定义配置及数据存放路径（通过 `.config_redirect` 实现）。
    - **定时自动保存设置**：启用/禁用开关和保存间隔（1~60 分钟），控制周期性自动保存行为。
    - **用户字典管理**：
        - 字典条目按优先级从上到下排列，新添加的条目插入顶部（最高优先级）。
        - 提供「上移」「下移」按钮调整条目优先级顺序。
        - 支持导入 RL 字典文件（格式：`原文\t注音1,注音2...`，`＋` 代表连词），自动解析并翻译为内部字典格式。
        - 字典条目独立存储在 `dictionary.json`，若文件不存在，首次启动自动使用内置 RL 字典（1757 条）初始化。重置配置不影响字典数据。

### 5. KaraokePreview (卡拉OK预览)
打轴界面内嵌的实时走字预览组件。

- **职责**：
    - 逐字高亮渲染，支持 Ruby 注音同步走字。
    - 无时间戳字符连读渲染（按邻近组时间均分）。
    - 连词组合并渲染 Ruby 和连词框。
    - **预计算偏移渲染**：使用 `ch.render_timestamps` 和 `ch.render_sentence_end_ts` 进行渲染，消除双重偏移逻辑。
    - **渲染缓存优化**：每帧复用已缓存的逐句渲染数据（字符宽度、分组、wipe时间、连词组信息），仅在数据变更时重新计算，显著降低 60fps 下的 CPU 开销。
