# UI 层设计

StrangeUtaGame 提供了三套主要的编辑界面，分别针对打轴、行编辑和全文本编辑任务。

## 核心界面

### 1. EditorInterface (打轴界面)
这是主要的时间标签编辑界面。

- **职责**：
    - 提供类似节奏游戏的打轴体验。
    - 实时预览卡拉OK走字效果 (KaraokePreview)。
    - 全键盘驱动，支持高频率的时间标签记录。
    - **核心特性**：
        - 逐字高亮渲染，支持 Ruby 同步走字。
        - **无时间戳字符连读渲染**：支持有 checkpoint 但无时间戳的字符，参照邻近组的时间进行连续平滑渲染。
        - 连词 (Linked Words) 渲染。
        - 实时应用时间偏移。

### 2. EditInterface (行编辑界面)
提供基于表格的项目全局视图。

- **职责**：
    - 展示所有歌词行及其元数据（行号、演唱者、连词标记等）。
    - **行级操作**：添加行、删除行、复制行、插入行，支持多行选择。
    - **键盘快捷键**：Ctrl+C 复制行、Ctrl+V 粘贴行、Delete 删除行。
    - **LineDetailDialog**：双击行打开详细编辑对话框，可逐个 Character 编辑注音、节奏点和演唱者。
    - **字符级操作**：在行详情中支持添加、删除、复制、插入字符，Ctrl+C/V/Delete 快捷键。
    - 操作后自动维护领域不变量（is_line_end、check_count、linked_to_next）。

### 3. RubyInterface (全文本编辑界面)
专注于注音编辑和文本调整。

- **职责**：
    - **全类型注音**：自动分析为所有字符类型（汉字、平假名、片假名、英文、数字、符号、空格）生成注音。
    - 用户可通过「按类型删除注音」按钮按字符类型批量清除不需要的注音（含長音符号ー/～、特殊符号♪等）。
    - 手动编辑注音文本及其对应的字符范围。
    - **自由文本编辑**：支持直接修改歌词行文本，系统自动维护已有的时间标签和注音。
    - **自动应用修改**：切换标签页离开全文本编辑时自动将修改应用回项目数据，无需手动点击按钮。
    - **注音保留**：文本未变更时仅更新标注的注音，不清除未标注字符的已有注音；文本变更时通过 SequenceMatcher 映射保留匹配字符的注音。
    - **更新节奏点**：根据用户调整后的注音数据和 AutoCheck 设置规则重新计算节奏点。

## 辅助界面
- **MainWindow**：基于 FluentWindow 的主框架，包含侧边栏导航。全局 Ctrl+S 快捷键保存项目（已有路径直接保存，无路径弹出另存为对话框）。切换标签页时自动应用全文本编辑界面的修改。
- **StartupInterface**：项目创建向导，负责初始歌词导入和音频选择。
- **HomeInterface**：主页，管理最近音频和快捷入口。

### 4. SettingsInterface (设置界面)
管理全局配置与校准工具。

- **职责**：
    - **Offset 校准（可视化弹窗）**：
        - 点击「开始校准」弹出独立校准窗口。
        - 深色画布上 2 个白色滑块按 BPM 从左到右匀速滑动，每个滑块穿过中央红色判定线对应一拍，接近判定线时自动放大。
        - 滑块到达判定线时播放节拍器点击音（后台线程非阻塞播放）。
        - 用户按空格键敲击，采用区间二等分法判定最近的 beat 并计算偏移量。
        - 左上角实时显示「最近偏移」和「平均偏移」。
        - 右上角「重置」和「应用」按钮，应用后写入全局 Offset 设置。
        - BPM 可在校准过程中自由调节（60-240）。
    - **Auto Check 设置**：新增小写假名 check 开关、空格视为句尾开关，控制自动分析逻辑。

### 5. KaraokePreview (卡拉OK预览)
打轴界面内嵌的实时走字预览组件。

- **职责**：
    - 逐字高亮渲染，支持 Ruby 注音同步走字。
    - 无时间戳字符连读渲染（按邻近组时间均分）。
    - 连词组合并渲染 Ruby 和连词框。
    - **渲染缓存优化**：每帧复用已缓存的逐句渲染数据（字符宽度、分组、wipe时间、连词组信息），仅在数据变更时重新计算，显著降低 60fps 下的 CPU 开销。
