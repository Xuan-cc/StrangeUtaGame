# UI 层设计

## 概述

UI 层基于 **PyQt6 + PyQt-Fluent-Widgets**，负责用户界面展示和输入处理。本层**不包含业务逻辑**，所有业务操作委托给应用服务层。

## 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| GUI 框架 | PyQt6 | 成熟稳定，功能完善 |
| 组件库 | PyQt-Fluent-Widgets | 现代化 Fluent Design 风格 |
| 样式方案 | QSS | 类似 CSS，易于维护 |
| 图表/波形 | 自定义绘制 | 低延迟，高度可控 |

## 核心组件

### MainWindow

**职责**：
- 应用主容器
- 导航管理
- 子界面切换
- 全局快捷键处理

**启动流程**：
1. 应用启动时显示 StartupInterface
2. 用户输入/导入歌词并选择音频
3. 点击"创建项目"后切换到 EditorInterface
4. 左侧导航栏在 StartupInterface 时隐藏或禁用

**与下层交互**：
- 持有 ApplicationContext（包含各 Service）
- 响应 Service 回调更新界面

- 歌词初始输入和导入
- 音频文件选择
- 项目创建入口
- **打开已有项目**

**布局**：


**核心功能**：

1. **歌词输入**
   - 直接粘贴：从剪贴板粘贴任意文本
   - 文件导入：支持 TXT（纯文本）、LRC（含时间标签）、KRA（卡拉OK格式）
   - 拖拽导入：拖放文件到指定区域

2. **智能预处理**
   - 自动分割字符（日文/英文/符号）
   - 自动分析注音
   - 自动计算检查数（节奏点数量）
   - 识别句尾（行末字符）

3. **预览与修正**
   - 显示解析后的歌词结构
   - 显示每个字符的检查数
   - 显示句尾标记
   - 支持在预览中快速编辑

4. **音频选择**
   - 选择音频文件（MP3, WAV, FLAC, AAC 等）
   - **支持直接拖拽音频文件到区域**
   - 验证音频有效性
   - 显示音频基本信息

5. **创建项目**
   - 验证歌词和音频都已提供
   - 生成 SUG 项目文件
   - 切换到 EditorInterface

6. **打开已有项目（新增）**
   - 选择 .sug 项目文件
   - 解析项目数据（歌词 + 时间标签）
   - **弹出音频选择对话框**（因为 SUG 不存储音频路径）
   - 用户选择/拖入音频
   - 验证音频时长（可选）
   - 切换到 EditorInterface

**与下层交互**：
- 调用 ProjectService.create_new_project()
- 调用 ProjectService.load_project()（新增）
- 调用 AutoCheckService 分析歌词
- 调用 FileParsers 解析导入的歌词文件

### EditorInterface

**职责**：
- 打轴主界面
- 键盘事件捕获
- 协调预览组件和播放控制

**主要区域**：

1. **KaraokePreview（歌词预览）**
   - 显示多行歌词
   - 字符级ワイプ效果（渐变色）
   - 根据播放位置高亮当前字符
   - 60fps 刷新

2. **TransportBar（播放控制）**
   - 播放/暂停/停止
   - 进度条（可拖拽跳转）
   - 当前时间显示
   - 播放速度调节（0.5x ~ 2.0x）

3. **TimelineWidget（时间轴）**
   - 音频波形显示
   - 时间标签位置标记
   - 播放位置游标
   - 支持缩放

**快捷键绑定**：

| 按键 | 功能 |
|------|------|
| **Space 按下** | **【核心】为当前 Checkpoint 记录开始时间，立即切换到下一个 Checkpoint** |
| **Space 长按** | **句尾字符专用：按下开始，抬起结束句尾** |
| F1-F9 | 功能键（可配置，默认与 Space 相同） |
| A | 播放/暂停 |
| S | 停止 |
| Z | 后退 5 秒（可配置） |
| X | 前进 5 秒（可配置） |
| Q | 减速 |
| W | 加速 |
| ESC | 清除当前时间标签 |
| **+** | **在当前字符后添加节奏点（Checkpoint）** |
| **-** | **删除当前字符的最后一个节奏点** |
| **L** | **切换当前字符的句尾标记（Line End）** |
| **双击 Checkpoint** | **跳转到该时间前 X 秒（用于预听，X 可配置）** |
| **Alt + 滚轮** | **缩放界面（放大/缩小歌词显示）** |

**基础打轴交互说明**：

**Checkpoint 定义**：
- Checkpoint 表示"该字符开始播放"的时间点
- 当音频播放到 Checkpoint 的时间点时，对应的字符应该开始显示（高亮）

**普通字符打轴流程（Space 按下即完成）**：
1. 用户听到该唱某个字了
2. 按下 Space 键的瞬间，记录当前音频时间为该字符 Checkpoint 的时间
3. **立即**自动移动到下一个 Checkpoint
4. 用户无需抬起 Space 键，直接准备打下一个字

**示例**：
```
歌词：赤い花
Checkpoint：赤(1) い(1) 花(1)

播放进度：
[10.0秒] 用户听到"赤" → 按下 Space → 记录"赤"的时间为 10.0秒 → 光标自动跳到"い"
[10.5秒] 用户听到"い" → 按下 Space → 记录"い"的时间为 10.5秒 → 光标自动跳到"花"
[11.0秒] 用户听到"花" → 按下 Space → 记录"花"的时间为 11.0秒 → 完成
```

**关键点**：
- **按下即记录，无需等待抬起**
- **记录后立即跳转**，实现连续快速打轴
- 句尾字符除外（需要长按记录拖音）

### KaraokePreview 设计

**布局策略（文本编辑器模式）**：

**多行显示**：
- **不是仅显示当前播放的一句**，而是像文本编辑器一样显示多行歌词
- 显示范围：当前 Checkpoint 所在行及其前后若干行（如 ±3 行）
- 允许用户看到上下文：上一句和下一句歌词
- 滚动：自动滚动保持当前 Checkpoint 在可视区域中央

**缩放功能**：
- **Alt + 滚轮**：放大/缩小歌词显示
- 缩放范围：50% ~ 200%
- 仅缩放歌词区域，不影响控件大小
- 记住用户偏好

**ワイプ效果**：

- **未唱字符**：灰色
- **已唱字符**：高亮颜色（红色/青色）
- **正在唱字符**：从左到右渐变

**实现方式**：
- 每个字符独立 QLabel
- 自定义 paintEvent 实现渐变
- 使用裁剪区域实现部分着色
- 多行布局使用 QVBoxLayout + QHBoxLayout 组合

**刷新策略**：
- 位置变化时更新
- 60fps 刷新率
- 仅重绘变化的字符

**交互设计**：

**1. 双击 Checkpoint 跳转（预听功能）**

**用途**：快速跳转到要打轴的位置前 X 秒，提前听一下节奏

**操作**：
- 在 KaraokePreview 中双击某个 Checkpoint（节奏点标记）
- 系统自动：
  1. 获取该 Checkpoint 已记录的时间（如果已有时间标签）
  2. 或获取该 Checkpoint 的预估时间（基于前后已有时间标签插值）
  3. 跳转到：该时间 - X 秒（X 可配置，默认 2 秒）
  4. 自动播放

**配置项（在 TimingSettingsPanel 中）**：
- `double_click_pre_listen_seconds`：双击预听提前量（默认 2 秒，范围 0.5 ~ 10 秒）

**示例**：
```
当前状态：
赤[●]い花が咲いた      ← [●] 表示已打轴的 Checkpoint
    ↑
    正在打这个位置

用户双击下一个 Checkpoint（い）：
假设「い」的预估时间是 00:15.000
设置 X = 2 秒
跳转位置：00:15.000 - 00:02.000 = 00:13.000
音频跳转到 13 秒开始播放，用户听到「赤」后开始按空格打「い」
```

**2. 快速添加/删除 Checkpoint（节奏点）**

**添加 Checkpoint（+ 键）**：
- **场景**：自动检查计算的节奏点数量不够，需要手动添加
- **操作**：按 **+** 键
- **效果**：
  1. 在当前字符后插入一个新的 Checkpoint
  2. 当前字符的 check_count + 1
  3. 自动分配新的节奏点索引
  4. 光标移动到新添加的 Checkpoint
- **注意**：仅调整节奏点配置，不影响已打的时间标签（除非已有时间标签数量超过新的 check_count）

**删除 Checkpoint（- 键）**：
- **场景**：节奏点太多，需要减少
- **操作**：按 **-** 键
- **效果**：
  1. 删除当前字符的最后一个 Checkpoint
  2. 当前字符的 check_count - 1
  3. 如果该 Checkpoint 已有时间标签，一并删除
  4. 如果当前光标在该 Checkpoint，移动到前一个
- **约束**：每个字符至少保留 1 个 Checkpoint（不能全部删除）

**3. 句尾字符长按打轴（特殊交互）**

**设计目的**：句尾字符通常有拖音或长音，需要记录开始和结束两个时间点

**操作方式**：
1. **按下 Space**：记录句尾字符的开始时间（第一个 Checkpoint）
   - 此时字符开始高亮（ワイプ效果开始）
   - 保持按下状态，表示"正在唱这个句尾"
2. **保持按下**：用户听到句尾唱完（拖音结束）
3. **抬起 Space**：记录句尾字符的结束时间（第二个 Checkpoint）
   - 此时ワイプ效果完成
   - 自动切换到下一行的第一个字符

**技术实现**：
- 检测键盘按下（KeyDown）和抬起（KeyUp）事件
- KeyDown：如果当前字符是句尾（is_line_end = true），记录开始时间
- KeyUp：如果已记录开始时间，则记录结束时间，并跳转下一行

**视觉反馈**：
- 按下期间：句尾字符持续高亮（或显示进度条表示拖音进度）
- 抬起后：ワイプ完成，切换到下一行

**示例**：
```
歌词：  赤い花が咲いた
句尾：          ↑「た」是句尾

用户操作：
1. 播放到「た」时按下 Space
   - 记录时间：00:20.000（句尾开始）
   - 「た」开始高亮
2. 保持 Space 按下，听到「た」拖音结束（约 0.5 秒）
3. 抬起 Space
   - 记录时间：00:20.500（句尾结束）
   - 「た」完全高亮
   - 自动跳到下一行「空が青い」的第一个字
```

**特殊情况处理**：
- 如果用户按下后很短时间就抬起（< 100ms）：视为普通点击，只记录一个时间点
- 如果用户长按超过 5 秒：自动结束（防止误操作）
- 如果用户切换了播放位置：取消当前的按下状态

**4. 快速切换句尾标记（L 键）**

**用途**：快速标记/取消标记句尾字符（影响渲染和导出）

**操作**：按 **L** 键（Line End）
- **当前字符标记为句尾**：设置 is_line_end = true
  - 视觉效果：字符后显示句尾标记（如「|」或特殊下划线）
  - 影响：导出时可能特殊处理（如添加休止时间）
- **当前字符取消句尾**：设置 is_line_end = false
- **切换**：如果已经是句尾则取消，如果不是则标记

**句尾标记的视觉反馈**：
- 句尾字符显示特殊标记（如竖线 | 或下划线）
- 颜色略有不同（如更深的颜色）
- 行末自动视为句尾（无需手动标记）

**快捷键配置**：
所有上述快捷键（+/-/L）可在 ShortcutSettingsPanel 中自定义

### SingerManagerInterface（演唱者管理界面）

**职责**：
- 配置演唱者属性（名称、颜色）
- 添加/删除演唱者
- 分配歌词给演唱者
- **注意**：用户不需要在此界面"切换当前演唱者"，系统自动管理

**界面布局**：
```
SingerManagerInterface
├── SingerListPanel（演唱者列表）
│   ├── 演唱者列表
│   │   - 复选框（启用/禁用，禁用的演唱者不参与打轴）
│   │   - 颜色块（可点击修改，用于视觉区分）
│   │   - 名称（可编辑，如"初音ミク"、"合唱"）
│   │   - 歌词范围显示（如"第3-5行"）
│   │   - 删除按钮（至少保留一个）
│   └── 操作按钮
│       - "添加演唱者"（自动生成颜色，用于和声）
│
└── SingerDetailPanel（演唱者详情）
    ├── 名称输入框
    ├── 颜色选择器
    ├── 歌词范围配置
    │   - 起始行 / 结束行
    │   - 或：从文本中选择段落分配给该演唱者
    ├── 统计信息
    │   - 歌词行数
    │   - 已打轴/未打轴字符数
    └── 操作按钮
        - "清空歌词"（删除该演唱者的所有歌词）
        - "合并到其他演唱者"（转移后删除该演唱者）
```

**交互设计**：
- **仅配置**：此界面仅用于配置演唱者属性，不用于"切换当前"
- 点击列表项：查看/编辑该演唱者的配置
- 拖拽选择文本：在导入歌词时分配段落给演唱者

**使用场景**：
- **启动时**：默认只有"演唱者1"，无需管理
- **添加和声时**：
  1. 在 StartupInterface 输入完整歌词
  2. 选择主歌部分分配给演唱者1

### KaraokePreview 多演唱者渲染设计

**职责**：
- 同时渲染多个演唱者的歌词
- 根据播放位置高亮每个演唱者的当前字符
- 使用时间异常检测和特殊渲染

**渲染策略**：

#### 1. 自动演唱者模式（默认）
```
┌─────────────────────────────┐
│ 赤い花が咲いた                │ ← 自动显示当前 Checkpoint 所属演唱者
│     ↑（高亮，颜色自动变化）   │
└─────────────────────────────┘
```
- **核心设计**：用户不需要知道当前是哪个演唱者
- 系统自动根据当前 Checkpoint 位置显示对应演唱者
- 演唱者切换平滑，用户感知为"连续往下打"
- 使用演唱者颜色进行视觉区分
- **示例流程**：
  1. 当前是演唱者1的 Checkpoint → 显示演唱者1歌词（红色）
  2. 用户打完，下一个是演唱者2的 Checkpoint → 自动切换到演唱者2歌词（蓝色）
  3. 用户继续按空格，无感知切换

#### 2. 播放时和声渲染（自动）
```
┌─────────────────────────────┐
│ 赤い花が咲いた                │ ← 主唱（演唱者1，红色）
│     ↑                        │
│ 青い空が広がる                │ ← 和声（演唱者2，蓝色）
│     ↑                        │ ← 同时高亮
└─────────────────────────────┘
```
- **播放时自动检测**：同一时间点多个演唱者有歌词
- **自动并排渲染**：垂直排列，不同颜色区分
- **用户无需切换**：播放时自动显示所有演唱者
- **打轴时专注**：打轴时只显示当前 Checkpoint 对应的演唱者

#### 3. 时间异常渲染（用户修改导致的时间倒退）

**检测逻辑**：
- 仅在同演唱者内检测时间顺序
- 当后一个时间标签早于前一个时，标记为"异常"

**渲染方式**：
- **播放前**：正常显示，但时间标签位置显示警告图标
- **播放中**：正常高亮
- **播放后**（该段已唱完）：
  - 异常段落高亮显示（如红色背景闪烁）
  - 显示警告提示："时间异常，请调整"
  - 点击可快速跳转到该位置

**示例**：
```
时间点 1: [10秒] 赤（正常，绿色）
时间点 2: [12秒] い（正常，绿色）
时间点 3: [11秒] 花（异常，红色闪烁）← 11秒比前一个12秒早
时间点 4: [15秒] が（正常，等待播放）
```

**不阻止打轴**：
- 用户可以继续打后面的轴点
- 系统仅渲染警告，不弹出阻塞提示
- 用户完成后统一调整异常段落

#### 4. 和声场景渲染（重点）

**场景**：两个演唱者同时唱不同歌词

```
时间点: [10秒]
演唱者1: 赤い花（"赤い"正在唱，高亮红色）
演唱者2: 青い空（"青い"正在唱，高亮蓝色）
        ↑ 同时高亮，不冲突
```

**渲染实现**：
- 每个演唱者独立的渲染上下文
- 独立计算每个演唱者的当前字符位置
- 使用不同颜色区分（如红/蓝/绿/黄）
- 歌词行垂直排列，避免重叠

**颜色分配**：
- 演唱者1：红色 (#FF6B6B)
- 演唱者2：蓝色 (#4ECDC4)
- 演唱者3：绿色 (#95E1D3)
- 演唱者4：黄色 (#FCE38A)
- 更多演唱者：循环或用户自定义

#### 5. 播放时渲染逻辑（详细）

**基本渲染规则**：
- **未开始**：字符显示灰色（#888888）
- **正在唱**：字符从左到右渐变（已唱部分高亮颜色，未唱部分灰色）
- **已唱完**：字符完全高亮（演唱者颜色）

**渲染计算（每帧 60fps）**：

**1. 普通字符渲染（单个 Checkpoint）**
```
字符：「赤」
Checkpoint：[10.0秒]

播放时间：10.0秒 → 开始高亮（左边缘）
播放时间：10.5秒 → 渐变到 50%
播放时间：11.0秒（下一个 Checkpoint）→ 完全高亮
```

**渐变计算**：
- 起始时间 = Checkpoint 时间
- 结束时间 = 下一个 Checkpoint 时间（或默认 +0.5秒）
- 进度 = (当前时间 - 起始时间) / (结束时间 - 起始时间)
- 进度 0% ~ 100% 对应从左到右渐变

**2. 连打字符渲染（多个 Checkpoint）**
```
字符：「赤」
Checkpoint：[10.0秒] [10.2秒] （あ・か）

播放时间：10.0秒 → 开始高亮「あ」
播放时间：10.2秒 → 「あ」完全高亮，开始「か」
播放时间：10.4秒 → 「か」完全高亮
```

**3. 句尾拖音渲染（长按打轴的特殊处理）**
```
字符：「た」（句尾，有开始和结束两个 Checkpoint）
Checkpoint：[20.0秒, 20.8秒]

播放时间：20.0秒 → 按下 Space 记录的开始时间
播放时间：20.0 ~ 20.8秒 → 持续渐变（进度从 0% 到 100%）
播放时间：20.8秒 → 抬起 Space 记录的结束时间，完全高亮

特殊效果：
- 拖音期间显示进度条或光晕效果
- 可选显示拖音时长（如 "0.8s"）
```

**4. 连打字符（check_count = 0）渲染**
```
字符：「ー」（长音，check_count = 0，和上一个字符连唱）
前一个字符：「か」[10.2秒]

渲染逻辑：
- 「か」开始高亮时（10.2秒），「ー」也开始高亮
- 「か」完全高亮时，「ー」也完全高亮
- 两者同步，视为一个整体
```

**5. 多演唱者播放渲染（和声）**

**独立计算每个演唱者**：
```
时间点：[10.0秒]
演唱者1：「赤」Checkpoint [10.0秒] → 开始高亮（红色）
演唱者2：「青」Checkpoint [10.0秒] → 开始高亮（蓝色）

计算逻辑：
- 对每个演唱者独立计算当前字符和进度
- 演唱者1：计算在演唱者1的时间标签中的位置
- 演唱者2：计算在演唱者2的时间标签中的位置
- 两者独立渲染，颜色不同
```

**6. 时间异常渲染**

**检测**：播放时检测时间标签顺序
```
演唱者1：
[10.0秒]「赤」→ 正常
[12.0秒]「い」→ 正常
[11.0秒]「花」→ 异常（11.0 < 12.0）

播放时间：11.0秒时：
- 「赤」已唱完（绿色）
- 「い」正在唱（进度约 50%，绿色）
- 「花」也开始高亮（红色闪烁，表示异常）

异常标识：
- 时间倒退的字符显示红色边框或闪烁
- 播放完成后显示警告标记
```

**7. 未打轴字符渲染**
```
歌词：「赤い花」
已打轴：「赤」[10.0秒]「い」[10.5秒]
未打轴：「花」（无 Checkpoint 时间）

播放时间：10.5秒后 → 「花」显示为未打轴状态
未打轴状态：
- 颜色：灰色（或半透明）
- 标记：显示 "?" 或特殊图标
- 进度：不计算渐变，保持未开始状态
```

**渲染优化**：
- 仅计算当前可视区域内的字符（虚拟列表）
- 使用双缓冲避免闪烁
- 缓存渲染结果（时间标签未变更时复用）

## 与下层交互

UI 层通过调用 Application Service 方法执行业务操作：

```
TimingService.on_timing_key("SPACE")
```

服务层通过**回调函数**通知 UI 状态变更：

```
TimingService 添加时间标签
    ↓
调用 on_timetag_added 回调
    ↓
EditorInterface._on_timetag_added()
    ↓
更新 KaraokePreview 显示
```

**回调类型**：
- 数据变更回调（时间标签添加、位置变化）

- 错误回调（操作失败）

### 避免直接依赖

**原则**：
- UI 不直接操作领域对象
- UI 不直接调用基础设施
- 所有操作通过 Service 层

### 测试场景

#### 场景 1: 打轴延迟测试

**测试方法**：
```python
def test_timing_latency():
    """测试打轴延迟"""
    timestamps = []
    
    for i in range(100):
        start = time.perf_counter_ns()
        # 模拟按键
        timing_service.on_timing_key("SPACE")
        end = time.perf_counter_ns()
        
        latency_ms = (end - start) / 1_000_000
        timestamps.append(latency_ms)
        
        time.sleep(0.1)  # 模拟正常打轴间隔
    
    avg_latency = sum(timestamps) / len(timestamps)
    max_latency = max(timestamps)
    
    assert avg_latency < 20, f"平均延迟 {avg_latency}ms 超过 20ms"
    assert max_latency < 50, f"最大延迟 {max_latency}ms 超过 50ms"
```

### 性能优化策略

#### 1. 渲染优化

- **双缓冲**：所有自定义绘制使用双缓冲
- **局部重绘**：仅重绘变化的字符，而非整行
- **虚拟列表**：仅渲染可视区域内的歌词行
- **缓存渲染结果**：时间标签未变更时复用

#### 2. 音频优化

- **预加载**：音频数据加载到内存
- **环形缓冲区**：减少内存分配
- **异步 I/O**：大文件使用异步加载

#### 3. 计算优化

- **延迟计算**：时间异常检测在后台线程执行
- **批量更新**：批量操作时减少回调次数
- **空间分割**：歌词行使用空间索引加速查找

```python
# 性能分析装饰器
def profile(func):
    """性能分析装饰器"""
    def wrapper(*args, **kwargs):
        profiler = cProfile.Profile()
        profiler.enable()
        
        result = func(*args, **kwargs)
        
        profiler.disable()
        stats = pstats.Stats(profiler)
        stats.sort_stats('cumulative')
        stats.print_stats(20)  # 打印前 20 个耗时函数
        
        return result
    return wrapper

# 帧率监控类
class FPSMonitor:
    """帧率监控器"""
    
    def __init__(self, target_fps: int = 60):
        self.target_fps = target_fps
        self.frame_times: List[float] = []
        self.max_history = target_fps * 2  # 保留 2 秒的历史
    
    def tick(self) -> None:
        """记录一帧"""
        now = time.perf_counter()
        self.frame_times.append(now)
        
        # 清理旧数据
        cutoff = now - (self.max_history / self.target_fps)
        self.frame_times = [t for t in self.frame_times if t > cutoff]
    
    def get_fps(self) -> float:
        """获取当前帧率"""
        if len(self.frame_times) < 2:
            return 0.0
        
        duration = self.frame_times[-1] - self.frame_times[0]
        return len(self.frame_times) / duration if duration > 0 else 0.0
    
    def is_performant(self) -> bool:
        """检查性能是否达标"""
        return self.get_fps() >= self.target_fps * 0.9
```
