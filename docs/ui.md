# UI 层设计

## 概述

UI 层基于 **PyQt6 + PyQt-Fluent-Widgets**，负责用户界面展示和输入处理。本层**不包含业务逻辑**，所有业务操作委托给应用服务层。

## 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| GUI 框架 | PyQt6 | 成熟稳定，功能完善 |
| 组件库 | PyQt-Fluent-Widgets | 现代化 Fluent Design 风格 |
| 样式方案 | QSS | 类似 CSS，易于维护 |
| 图表/波形 | 自定义绘制 | 低延迟，高度可控 |

## 核心组件

### MainWindow

**职责**：
- 应用主容器
- 导航管理
- 子界面切换
- 全局快捷键处理

**启动流程**：
1. 应用启动时显示 StartupInterface
2. 用户输入/导入歌词并选择音频
3. 点击"创建项目"后切换到 EditorInterface
4. 左侧导航栏在 StartupInterface 时隐藏或禁用

**与下层交互**：
- 持有 ApplicationContext（包含各 Service）
- 响应 Service 回调更新界面

### StartupInterface（启动界面）

**职责**：
- 歌词初始输入和导入
- 音频文件选择
- 项目创建入口

**布局**：

```
StartupInterface
├── 顶部：Logo + 标题
├── 左侧：LyricInputPanel（歌词输入区）
│   ├── 大文本编辑框
│   │   - 支持多行输入
│   │   - 支持直接粘贴
│   │   - 占位符："在此粘贴歌词，或拖拽歌词文件到此处..."
│   ├── 导入按钮组
│   │   - "从文件导入 (TXT/LRC/KRA)"
│   │   - "从剪贴板粘贴"
│   │   - "清空"
│   └── 文件拖拽区域
│       - 高亮显示拖放区域
│       - 支持 .txt, .lrc, .kra 文件
├── 中间：ImportPreview（导入预览）
│   ├── 歌词结构预览表
│   │   - 行号
│   │   - 文本内容
│   │   - 字符数
│   │   - 自动检查数（显示每个字的节奏点数量）
│   ├── 注音预览
│   │   - 显示自动生成的注音
│   │   - 可在此界面快速修正
│   └── 错误提示
│       - 空行警告
│       - 超长行警告
└── 右侧：AudioSelectPanel（音频选择区）
    ├── 音频文件选择
    │   - 文件路径输入框
    │   - 浏览按钮
    │   - 拖拽支持
    ├── 音频信息预览
    │   - 文件名
    │   - 时长
    │   - 格式
    └── 操作按钮
        - "创建项目并进入编辑器"（主按钮，条件启用）
        - "取消"
```

**核心功能**：

1. **歌词输入**
   - 直接粘贴：从剪贴板粘贴任意文本
   - 文件导入：支持 TXT（纯文本）、LRC（含时间标签）、KRA（卡拉OK格式）
   - 拖拽导入：拖放文件到指定区域

2. **智能预处理**
   - 自动分割字符（日文/英文/符号）
   - 自动分析注音
   - 自动计算检查数（节奏点数量）
   - 识别句尾（行末字符）

3. **预览与修正**
   - 显示解析后的歌词结构
   - 显示每个字符的检查数
   - 显示句尾标记
   - 支持在预览中快速编辑

4. **音频选择**
   - 选择音频文件（MP3, WAV, FLAC, AAC 等）
   - 验证音频有效性
   - 显示音频基本信息

5. **创建项目**
   - 验证歌词和音频都已提供
   - 生成 RLF 项目文件
   - 切换到 EditorInterface

**与下层交互**：
- 调用 ProjectService.create_new_project()
- 调用 AutoCheckService 分析歌词
- 调用 FileParsers 解析导入的歌词文件

### EditorInterface

**职责**：
- 打轴主界面
- 键盘事件捕获
- 协调预览组件和播放控制

**主要区域**：

1. **KaraokePreview（歌词预览）**
   - 显示多行歌词
   - 字符级ワイプ效果（渐变色）
   - 根据播放位置高亮当前字符
   - 60fps 刷新

2. **TransportBar（播放控制）**
   - 播放/暂停/停止
   - 进度条（可拖拽跳转）
   - 当前时间显示
   - 播放速度调节（0.5x ~ 2.0x）

3. **TimelineWidget（时间轴）**
   - 音频波形显示
   - 时间标签位置标记
   - 播放位置游标
   - 支持缩放

**快捷键绑定**：

| 按键 | 功能 |
|------|------|
| Space | 在当前位置打轴 |
| F1-F9 | 功能键（可配置） |
| A | 播放/暂停 |
| S | 停止 |
| Z | 后退 5 秒 |
| X | 前进 5 秒 |
| Q | 减速 |
| W | 加速 |
| ESC | 清除当前时间标签 |

### KaraokePreview 设计

**ワイプ效果**：

- **未唱字符**：灰色
- **已唱字符**：高亮颜色（红色/青色）
- **正在唱字符**：从左到右渐变

**实现方式**：
- 每个字符独立 QLabel
- 自定义 paintEvent 实现渐变
- 使用裁剪区域实现部分着色

**刷新策略**：
- 位置变化时更新
- 60fps 刷新率
- 仅重绘变化的字符

### TransportBar 设计

**播放控制**：
- 播放/暂停按钮（切换状态）
- 停止按钮（重置位置）
- 快捷键提示（括号内）

**进度显示**：
- 滑块：当前播放位置
- 标签：`MM:SS.CC` 格式（分:秒.厘秒）

**速度控制**：
- 滑块：50% ~ 200%
- 标签：显示当前倍率（如 1.0x）

### RubyInterface 设计

**注音编辑表**：
- 行：每行歌词
- 列：字符、注音、操作
- 支持批量编辑

**功能**：
- 自动分析注音
- 手动编辑注音
- 连字符号（+）表示连读

### ExportInterface 设计

**导出面板**：
- 格式列表（单选）
- 选项配置（因格式而异）
- **演唱者选择**：全部 / 指定演唱者
- 输出路径选择
- 导出按钮

**批量导出**：
- 支持一次导出多种格式
- 支持按演唱者分别导出（生成多个文件）
- 进度显示
- 错误提示

### SingerManagerInterface（演唱者管理界面）

**职责**：
- 添加/删除/重命名演唱者
- 设置演唱者颜色
- 切换当前演唱者
- 启用/禁用演唱者显示

**界面布局**：
```
SingerManagerInterface
├── SingerListPanel（演唱者列表）
│   ├── 演唱者列表（可拖拽排序）
│   │   - 复选框（启用/禁用）
│   │   - 颜色块（可点击修改）
│   │   - 名称（可编辑）
│   │   - "当前"标记
│   │   - 删除按钮（至少保留一个）
│   └── 操作按钮
│       - "添加演唱者"（自动生成颜色）
│       - "设为当前"（仅一个可设为当前）
│
└── SingerDetailPanel（演唱者详情）
    ├── 名称输入框
    ├── 颜色选择器
    ├── 统计信息
    │   - 歌词行数
    │   - 已打轴/未打轴字符数
    └── 操作按钮
        - "清空歌词"（保留演唱者）
        - "转移歌词"（转移到其他演唱者）
```

**交互设计**：
- 点击列表项：选中该演唱者（不切换当前）
- 双击列表项：设为当前演唱者并关闭界面
- 拖拽排序：调整演唱者优先级（影响显示顺序）

**使用场景**：
- **启动时**：默认只有"演唱者1"，无需管理
- **添加和声时**：点击"添加演唱者"，命名为"合唱"或"和声"
- **切换轨道时**：在列表中选择要编辑的演唱者，点击"设为当前"
- **导出时**：可选择导出全部演唱者或仅当前演唱者

### KaraokePreview 多演唱者渲染设计

**职责**：
- 同时渲染多个演唱者的歌词
- 根据播放位置高亮每个演唱者的当前字符
- 使用时间异常检测和特殊渲染

**渲染策略**：

#### 1. 单演唱者模式
```
┌─────────────────────────────┐
│ 演唱者1: 赤い花が咲いた      │ ← 仅显示当前演唱者
│           ↑（高亮红色）      │
└─────────────────────────────┘
```
- 仅渲染当前演唱者的歌词
- 全屏显示，字体较大
- 专注打轴，不受其他演唱者干扰

#### 2. 全部演唱者模式（和声场景）
```
┌─────────────────────────────┐
│ 演唱者1: 赤い花が咲いた      │
│           ↑（高亮红色）      │
│ 演唱者2: 青い空が広がる      │ ← 同时显示
│           ↑（高亮蓝色）      │
│ 演唱者3: （休止中）           │ ← 无歌词时显示占位
└─────────────────────────────┘
```
- 渲染所有启用的演唱者
- 每行显示演唱者名称和颜色标识
- 每个演唱者独立计算当前高亮字符
- 支持同时高亮（和声场景）

#### 3. 时间异常渲染（用户修改导致的时间倒退）

**检测逻辑**：
- 仅在同演唱者内检测时间顺序
- 当后一个时间标签早于前一个时，标记为"异常"

**渲染方式**：
- **播放前**：正常显示，但时间标签位置显示警告图标
- **播放中**：正常高亮
- **播放后**（该段已唱完）：
  - 异常段落高亮显示（如红色背景闪烁）
  - 显示警告提示："时间异常，请调整"
  - 点击可快速跳转到该位置

**示例**：
```
时间点 1: [10秒] 赤（正常，绿色）
时间点 2: [12秒] い（正常，绿色）
时间点 3: [11秒] 花（异常，红色闪烁）← 11秒比前一个12秒早
时间点 4: [15秒] が（正常，等待播放）
```

**不阻止打轴**：
- 用户可以继续打后面的轴点
- 系统仅渲染警告，不弹出阻塞提示
- 用户完成后统一调整异常段落

#### 4. 和声场景渲染（重点）

**场景**：两个演唱者同时唱不同歌词

```
时间点: [10秒]
演唱者1: 赤い花（"赤い"正在唱，高亮红色）
演唱者2: 青い空（"青い"正在唱，高亮蓝色）
        ↑ 同时高亮，不冲突
```

**渲染实现**：
- 每个演唱者独立的渲染上下文
- 独立计算每个演唱者的当前字符位置
- 使用不同颜色区分（如红/蓝/绿/黄）
- 歌词行垂直排列，避免重叠

**颜色分配**：
- 演唱者1：红色 (#FF6B6B)
- 演唱者2：蓝色 (#4ECDC4)
- 演唱者3：绿色 (#95E1D3)
- 演唱者4：黄色 (#FCE38A)
- 更多演唱者：循环或用户自定义

## 与下层交互

### 调用应用服务

UI 层通过调用 Application Service 方法执行业务操作：

```
用户按空格键
    ↓
EditorInterface._on_space_pressed()
    ↓
TimingService.on_timing_key("SPACE")
    ↓
业务逻辑执行...
```

### 响应服务回调

服务层通过**回调函数**通知 UI 状态变更：

```
TimingService 添加时间标签
    ↓
调用 on_timetag_added 回调
    ↓
EditorInterface._on_timetag_added()
    ↓
更新 KaraokePreview 显示
```

**回调类型**：
- 数据变更回调（时间标签添加、位置变化）
- 状态变更回调（播放/暂停、项目加载）
- 错误回调（操作失败）

### 避免直接依赖

**原则**：
- UI 不直接操作领域对象
- UI 不直接调用基础设施
- 所有操作通过 Service 层

## 状态管理

### UI 本地状态

- 当前激活的界面
- 窗口大小/位置
- 用户偏好设置（主题、字体等）

### 业务状态

- 当前项目数据：存储在 ProjectService
- 打轴状态：存储在 TimingService
- UI 只读取，不直接修改

### 状态同步

- 通过回调机制同步
- 避免轮询
- 批量更新减少重绘

## 性能优化

### 渲染优化

- **双缓冲**：避免绘制闪烁
- **局部重绘**：仅更新变化的区域
- **硬件加速**：利用 GPU 加速（Qt 自动）

### 大文件处理

- **虚拟列表**：仅渲染可见的歌词行
- **波形缓存**：预生成或按需生成
- **延迟加载**：非激活界面不渲染

### 响应速度

- **异步操作**：文件 I/O 不阻塞 UI
- **进度反馈**：长时间操作显示进度
- **取消机制**：支持取消耗时操作

## 样式设计

### 主题

- **暗色主题**：默认，适合长时间使用
- **亮色主题**：可选
- **强调色**：蓝色（#0078D4）

### 字体

- **界面字体**：Microsoft YaHei / PingFang SC
- **歌词字体**：大号（24-32px），清晰可读
- **等宽字体**：时间显示

## 错误处理

### 用户确认

- **覆盖确认**：文件已存在
- **放弃确认**：有未保存更改
- **删除确认**：删除时间标签等
